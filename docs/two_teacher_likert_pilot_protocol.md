# 双教师 1-5 分问卷标注协议

## 目的

本协议用于收集英语教师对 AI 生成 ESL 写作反馈的人工判断，从而评估
ConsensusScope 的反馈路由是否能把高风险或不适合直接给学生的反馈送入人工复核队列。

该标注只用于研究分析，不是自动作文评分，不是课堂成绩评定，也不是教师替代系统。

## 标注人员

- 最多两位英语教师参与。
- 系统中教师编号固定为 `1` 或 `2`。
- 每位教师可选择批次编号 `1` 或 `2`，用于区分不同标注批次。

## 数据要求

- 只使用匿名化学习者英语作文和 AI 反馈。
- 不上传姓名、学号、邮箱、班级、学校、联系方式或其他可识别个人身份的信息。
- 如果使用公开学习者数据集，需遵守数据集许可和引用要求。

## 标注流程

正式标注只使用盲标模式：

```text
开始标注 -> 逐条反馈 1-5 分问卷 -> 查看进度 -> 导出结果
```

盲标模式不会显示系统风险等级、推荐动作、模型一致性、模型名称或
ConsensusScope 决策，避免教师判断受到系统输出影响。

## 问卷维度

每条 AI 反馈按六个维度打 1-5 分：

- 正确性：这条反馈本身是否正确。
- 保留原意：这条反馈是否保留学生原本想表达的意思。
- 学生可见性：这条反馈是否适合直接给学生看。
- 修改有用性：这条反馈是否能帮助学生修改作文。
- 清晰可操作性：这条反馈是否清楚、具体、可执行。
- 直接放行：这条反馈是否可以不经教师复核直接使用。

评分锚点：

```text
1 = 明显负面 / 不安全 / 不适合直接给学生
3 = 不确定
5 = 明显正面 / 安全 / 可以直接使用
```

正式 pilot 不要求教师写理由，不要求选择复杂类别标签。

## 分析指标

导出 `likert_feedback_ratings.csv` 后，可运行：

```bash
PYTHONPATH=. python3 scripts/analyze_teacher_likert_pilot.py \
  --ratings path/to/exported_teacher_ratings \
  --routing data/esl_writing_demo/routing_results.csv
```

脚本输出：

- 每条反馈在六个维度上的教师平均分；
- 两位教师的一致性，包括 exact agreement、within-1-point agreement 和 Spearman 相关；
- 系统自动放行比例和人工复核比例；
- 教师认为需要复核的反馈中，被系统送入复核队列的比例；
- 教师认为明显不安全的反馈中，被系统送入复核队列的比例；
- 系统自动放行反馈中，教师也认为可安全放行的比例。

## 论文写法边界

教师问卷结果是 offline diagnostic labels，只能用于事后评估系统路由质量。
论文中不能写成系统在部署时已经知道教师评分。

如果只有少量教师样本，应表述为 pilot validation 或 preliminary teacher-aligned
diagnosis，而不是大规模人工实验。
